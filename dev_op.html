DevOp Guidelines

Basic Guidelines

Performance Sheriff

  • Graph Spikes Handling
  • Missing Data Handling
  • Troubleshooting:
    • Test Cases Fixes: figure out basic test cases errors and fix it, including OCR, test scripts, and so on
    • Machines issues: figure out if machine slows down, crashes, and so on
    • Anything: all the issues related with operation or performance should be solved

Framework Quality Sheriff

  • Verify Pull Requests: make sure pull requests will be okay after merged
  • Verify Releases: detailed check and verification before releasing

Operation Specialist

  • Check Machine Status
    • Freespaces: keep 2 weeks of data only and keep at least 20% free space
    • Network: check if all network status are good and smooth
    • Restart computer: restart computers every week
  • Creating Documentation: document knowledge and share with publicity
  • Requests: document knowledge and share with publicity

Detailed Information

Graph Spikes Handling

  • Check Machines: check if there is crashes, windows updates, hardware issues, and so on
  • Check Historic Data: check if there is any strange things on historic data
  • Check Images & Videos: see if it's anything related with threshold or recording software
  • Validate Test Cases: make sure the test cases still fit current application under test
  • Produce Report: Oral report or written report is needed

Missing Data Handling

  • Check Machines: check if there is crashes, windows updates, hardware issues, and so on
  • Check Images & Videos: see if it's anything related with threshold or recording software
  • Validate Test Cases: make sure the test cases still fit current application under test
  • Check Free Spaces: if free space is less than 20%, the machine speed might decrease
  • Data Filling: fill up daily data with normal data
